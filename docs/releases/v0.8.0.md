# snor-oh v0.8.0 ‚Äî The Pet Carries Your Stash

*Released 2026-04-22*

Two user-visible rolls this release: the mascot finally reacts to the bucket in its care (Epic 02), and the bucket gains inline transforms with on-device OCR and translation (Epic 07). Both stayed strictly local ‚Äî nothing networks, no API keys, no Cloud tax.

## Highlights

### ü™Ñ The pet actually carries your stuff now

Up until v0.7 the bucket and the mascot lived in separate worlds. They don't anymore.

- **Inventory badge.** A small orange capsule with the active item count sits on the mascot's top-right corner. `99+` above 99, hidden at 0.
- **`.carrying` status.** When Claude Code isn't busy and the bucket has items, the pet switches into a "carrying" pose (orange dot, re-uses idle sprite per pet ‚Äî custom ohhs still work, `.carrying` is optional in the sprite table).
- **Catch animation.** Every new bucket item triggers a 400 ms orange tint flash on the mascot. Mascot-origin drops (dropping directly *onto* the pet) get a brighter, warmer flash.
- **"I'm heavy!" bubble.** At 20 / 50 / 100 items the pet pops an actionable speech bubble. Click it to open the bucket panel. Clearing the bucket re-arms the threshold so a refilled bucket nudges again.

### üñºÔ∏è Quick actions on bucket items

Right-click any bucket item ‚Üí **Actions ‚ñ∏**:

| Action | What it does |
|---|---|
| **Resize image** (50 / 25 / 10%) | ImageIO high-quality thumbnail, preserves format + orientation |
| **Convert to** PNG / JPEG / HEIC | JPEG quality 0.85, HEIC uses Apple's modern format |
| **Strip metadata** | Drops EXIF / GPS / TIFF / IPTC / XMP in one pass |
| **Extract text** | Vision OCR, on-device, writes back to source *and* creates a new text item |
| **Translate to‚Ä¶** *(macOS 15+)* | Apple's `Translation.framework`, on-device, free, offline once downloaded |

Derived items land **right next to their source** in the bucket, not at the head ‚Äî so resized photos sit beside their originals for comparison, and OCR'd text sits under its screenshot.

### üîç Searchable screenshots

Type any word into the bucket search field. Every image in the active bucket gets OCR'd in the background (bounded 4-way parallelism via `TaskGroup`) and results show up in the same scroll view. Subsequent searches hit the cache ‚Äî zero-cost.

Three indexing modes in **Settings ‚Üí Search Indexing**:

- **Lazy** *(default)* ‚Äî OCR runs on first search, cached after. Best battery + wow moment.
- **Eager** ‚Äî OCR every screenshot as it lands. Best for "always searching" users.
- **Manual** ‚Äî never auto-OCR; only when you click *Extract text*. Best for battery-paranoid.

### üëÄ Provenance visualization

OCR'd text cards now *look* derived:

- Thumbnail becomes a mini view of the source photo.
- Corner badge: orange circle with `text.viewfinder` glyph for OCR, purple `character.bubble` for translation, blue for image ops (resize / convert / strip).
- Secondary line shows "from *screenshot.png*" instead of "42 chars".
- Right-click ‚Üí **Reveal Source** scrolls back to the original with a 1.5 s accent-coloured outline pulse.

### ‚å®Ô∏è Quick Look works for every item kind now

Click any card (accent outline confirms focus) ‚Üí press **Space** ‚Üí

- Files, folders, images ‚Äî native Finder Quick Look
- Text ‚Äî rendered as `.txt` with formatting preserved
- Rich text ‚Äî rendered as `.rtf` with bold/italic/colors
- URLs ‚Äî rendered as `.webloc` with title + URL + favicon
- Colors ‚Äî 128√ó128 PNG swatch of the hex

Temp files live under `NSTemporaryDirectory()/snor-oh-preview/` and are keyed by item UUID, so re-previewing the same item doesn't leak files.

## Compatibility + migration

- **No schema migration.** Every new field on `BucketItem` / `BucketSettings` is optional with a default. Older JSON decodes into v0.8.0 with the new fields set to `nil` / their default values.
- **`.snoroh` files stay portable.** `.carrying` is opt-out of the sprite surface (see `Status.spriteStatuses`). Older snor-oh / ani-mime builds keep importing v0.8.0 exports, and we import theirs without demanding a carrying asset.
- **macOS 14.0 still supported.** The Translate menu hides itself pre-macOS 15 rather than showing a disabled option ‚Äî the programmatic Translation API requires macOS 15.
- **HTTP server still on `:1425`** (moved in v0.7.0 to coexist with ani-mime on `:1234`).

## Known issues

- Apple's Translation download prompt for a language pair not yet installed currently appears the *first* time you translate; subsequent translations bypass it. If the download flow feels rough, it's Apple's system UI ‚Äî not ours.
- OCR on very large screenshots (>4K) can briefly stutter scroll during the first lazy-index pass. Bounded to 4 concurrent tasks to keep this manageable; if you hit noticeable lag, switch to Manual mode in Settings.
